# Research Summary: Native Markdown Notepad

Date: 2026-04-30

## Comparable Apps

MarkText is an open-source Markdown editor for Linux, macOS, and Windows that explicitly focuses on speed and usability. Useful inspiration: keep the editor simple, elegant, and cross-platform.

Ferrite is a newer native Markdown editor that leans hard into the exact positioning Markpad wants: no Electron, low memory, fast launch, source/rendered panes, and large-file friendliness. Useful inspiration: make performance part of the product, not just an implementation detail.

Verso frames Markdown editing as "Preview.app for `.md` files": open a plain file, read/edit, close. Useful inspiration: avoid lock-in and avoid turning the app into a heavy knowledge base too early.

Typora and Cogito show the polish bar for Markdown typography and writing flow. Useful inspiration: source and preview must both feel calm, readable, and modern.

Mousepad is the right restraint model. The Xfce docs describe its target as quickly editing text files, not becoming a development environment or plugin platform. Useful inspiration: add practical menus/settings without letting the app sprawl.

GNOME Text Editor validates autosaved drafts for unsaved files. Its help docs say unsaved files are automatically saved in a draft folder and exposed through the app rather than making the user manage temporary files. Useful inspiration: drafts are a primary workflow, not an emergency-only feature.

FeatherPad is a useful feature boundary: lightweight editor, session management, side-pane mode, auto-saving, and non-interrupting prompts. Useful inspiration: a permanent left rail and autosave can coexist with a compact editor.

## Stack Choice

Gio was chosen over Wails/Tauri/Electron-style stacks because this project wants a Go-first native app without a bundled browser runtime. Gio supports Linux, macOS, Windows, and WebAssembly, and its docs say it is designed around very few dependencies: platform windowing, input, and GPU drawing.

goldmark was chosen for Markdown parsing because it is pure Go, CommonMark compliant, extensible, and ships GFM extensions such as tables, strikethrough, task lists, and linkifying.

## Release Choice

Desktop builds use the Go toolchain directly. Gio's `gogio` is still useful for WASM and some platform packaging, but Linux AppImage and `.deb` are better handled through release scripts.

GitHub Actions should build on each OS rather than cross-compiling everything from Linux. Gio's Linux dependencies include Wayland, X11/xkbcommon, GLES/EGL, libXcursor, and related dev packages. Windows needs no extra default dependencies. macOS needs Xcode.
